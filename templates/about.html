Our Story & Mission

About KidneyHealth AI

Revolutionizing kidney disease diagnostics through the power of artificial intelligence and deep learning

Our Mission

AI-Powered Diagnostics for Better Outcomes

KidneyHealth AI is an advanced medical imaging system built to assist clinicians, nephrologists, and radiologists in detecting kidney conditions from CT scan images using deep learning technology.

Our mission is to bridge the gap between cutting-edge AI research and real-world clinical application — providing faster, more accessible preliminary diagnostics that support medical decision-making in resource-constrained environments.

By automating the classification of Cyst, Stone, Tumor, and Normal kidney states, we aim to help reduce diagnostic delay and support early intervention that saves lives.

The Science

Technology Behind the System

Built on proven deep learning architectures with medical imaging best practices.

VGG16 Transfer Learning

Our classifier is built on the VGG16 architecture pretrained on ImageNet, fine-tuned on a curated kidney CT dataset for domain-specific feature extraction.

Curated Medical Dataset

Trained on 12,000+ annotated CT images across four classes, with balanced augmentation pipelines to prevent class bias and improve generalization.

Flask REST API

The inference engine is exposed via a Flask REST API with CORS support, accepting base64-encoded images and returning structured JSON predictions.

Secure Pipeline

All image data is processed ephemerally — decoded, analyzed, and discarded. No patient data is retained, aligning with clinical data privacy requirements.

Pipeline

Detailed Workflow

From image upload to classification — a transparent look at every step.

1

CT Image Acquisition

Users upload an axial kidney CT scan image (JPG, PNG, JPEG) via the web interface. The system validates format and size before proceeding.

2

Base64 Encoding & Transmission

The image is encoded to base64 in the browser and sent securely to the Flask backend via a POST request to the /predict endpoint.

3

Preprocessing

The decoded image is resized to 224×224 pixels, converted to an array, and normalized (pixel values scaled to [0,1]) to match the model's expected input format.

4

CNN Inference

The preprocessed tensor passes through the loaded VGG16-based model (model.h5), producing a probability vector across all four kidney classes.

5

Classification & Confidence

argmax determines the predicted class, and max(raw_prediction) provides the confidence score. Results are mapped to human-readable labels.

6

JSON Response & UI Display

The prediction is returned as a structured JSON object [{"class": "...", "confidence": 0.XX}] and rendered dynamically on the classify page.

Technical Details

Technical Specifications

Model Architecture

VGG16 backbone with custom fully-connected classification head, batch normalization, and dropout regularization

Training Dataset

12,000+ annotated kidney CT images across Cyst, Stone, Tumor, and Normal classes with stratified splits

Classification Classes

4 classes: Normal (0), Cyst (1), Stone (2), Tumor (3) — identified via argmax on softmax output

Input Format

224×224 RGB images, normalized to [0,1] range, batched with np.expand_dims

Supported Formats

JPG, JPEG, PNG — with automatic resizing and format conversion via Keras preprocessing

API Framework

Flask REST API with Flask-CORS, JSON I/O, base64 image decoding, and structured error handling

Model File

Saved as artifacts/training/model.h5, loaded at app initialization via TensorFlow/Keras

Pipeline Orchestration

DVC-managed ML pipeline supporting reproducible training, evaluation, and deployment workflows
